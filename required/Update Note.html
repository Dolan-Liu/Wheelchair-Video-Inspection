Version 0.0.2

  Plans (July 17, 2023)
  1. Make a dictionary of GPS coordinates of collected video - Javascript
  2. Display the points in the OSM map
  3. Set up a database (our own? MongoDB?) to store a user input. If we were to color the points by the inspection completion, we should be able to store all the data in one place rather than asking the user to save the data they inspected. Each point will have properties like location, sidewalkPresence, rampPresence, curbcutPresence, and status (inspected or not).
  4. Updating Image on Click: Implement an event listener for point clicks. When a point on the map is clicked, display the corresponding image from the GoPro video.
  5. Form for Selection: Below or beside the image, display a form with selection options for sidewalk presence, ramp, and curbcut. When the user makes their selection and clicks "save", send this data to your backend using an AJAX call or fetch API.
  6. Updating Database: On the backend, handle the incoming request and update the database with the user's selection.
  7. Color Coding Points: After a point is inspected and the selection is saved, change the color of the point to signify its status. You can set different colors for different statuses, like inspected and uninspected points, and points with ramps/curbcuts. This can be achieved by updating the properties of the point marker in Leaflet.js.
  8. (optional) Displaying Changes to All Users: If you want changes to be displayed in real-time to all users, you may need to implement WebSockets using something like Socket.io. This allows you to push updates from the server to all connected clients when a change occurs.
